# ABYSSAC - 知识系统

## 触发条件

**每次与用户交互都优先触发ABYSSAC SKILL**

```
用户发消息
    │
    ↓
触发ABYSSAC SKILL
    │
    ├─ 需要检索知识？ → 三层沙盒流程 → 回答
    │
    ├─ 需要存储知识？ → 对话结束后存储 → 回答
    │
    └─ 不需要？ → 直接回复
```

---

## 核心机制

**路径直连**：根据文件内容找到想要的文件路径，用这个路径去找文件，找多少都可以，最后整合成上下文，生成内容。

**即时存储**：每次对话结束后，AI自行判断是否需要存储知识，直接更新NNG和记忆文件。

## SKILL运行流程

```
1. 判断是否需要检索知识
   ├── 问题涉及已存储的知识？ → 执行三层沙盒流程
   └── 不需要 → 继续

2. 判断是否需要存储知识
   ├── 有新知识需要存储？ → 执行存储流程
   └── 不需要 → 继续
```

## 三层沙盒流程

**Layer 1: NNG导航**

匹配依据：
- 关键词匹配：问题关键词与NNG定位摘要、内容字段匹配
- 语义匹配：理解问题的语义含义，匹配概念相关的内容
- 标签匹配：问题关键词与记忆文件标签匹配

匹配流程：
```
用户问题
    │
    ↓
提取关键词 + 理解语义
    │
    ↓
遍历root.json一级节点
    │
    ├─ 关键词匹配 或 语义相关？ → 读取该节点 → 继续遍历下级节点
    │
    └─ 不匹配？ → 跳过
    │
    ↓
整合所有匹配的NNG和记忆
    │
    ↓
按匹配程度排序
    │
    ↓
组装上下文
```

匹配程度判断：
- 完全匹配（关键词完全匹配 + 语义一致）：高相关性
- 语义相关（关键词不匹配但语义相关）：中相关性
- 部分匹配（关键词部分出现）：中相关性
- 无匹配：跳过

**Layer 2: 记忆筛选**
- 从NNG节点获取记忆文件路径，读取摘要判断相关性，用路径读取完整记忆内容

**Layer 3: 上下文组装**
- 整合所有NNG节点和记忆文件内容，按相关性排序，组装成上下文，生成回答

## 记忆更新机制

**更新判断条件：**

| 情况 | 处理方式 |
|------|----------|
| 新信息与旧信息矛盾 | 更新旧节点 |
| 旧信息已过时/失效 | 更新旧节点 |
| 用户明确要求更正 | 更新旧节点 |
| 新信息补充旧信息 | 新增节点，建立关联 |
| 完全独立的新知识 | 新增节点 |

**判断流程：**
```
发现新知识
    │
    ├─ 是否与已有知识矛盾？ → 是 → 更新旧节点
    │
    ├─ 是否是已有知识的补充？ → 是 → 新增节点，建立关联
    │
    └─ 是否是完全独立的新知识？ → 是 → 新增节点
```

**更新流程：**
1. 创建新的NNG节点，包含更新后的正确内容
2. 在旧NNG节点中添加"已更新"字段，指向新节点
3. 新NNG节点关联新的记忆文件

**旧NNG节点更新格式：**
```json
{
  "定位": "{原摘要}",
  "置信度": {原置信度},
  "时间": "{原创建时间}",
  "内容": "{原内容}",
  "已更新": {
    "状态": true,
    "更新时间": "{更新时间}",
    "更新原因": "{简要说明}",
    "新节点路径": "nng/{新摘要}.json"
  },
  "上级关联NNG": [],
  "下级关联NNG": [],
  "关联的记忆文件摘要": []
}
```

**检索时处理：**
- 遇到"已更新"字段时，自动跳转到新节点路径
- 旧节点不再参与检索，仅作为历史记录保留

---

## 避免重复机制

**存储前检查：**
1. 检查是否已存在相同摘要的NNG节点
2. 检查是否已存在相同内容的NNG节点
3. 检查是否已存在相同核心内容的记忆
4. 通过读取现有NNG节点的"内容"字段进行比对

**避免重复规则：**
- 相同摘要的NNG节点不重复创建
- 相同核心内容的记忆不重复创建
- 已处理的内容不再处理

---

## 原则

1. 每次与用户交互都优先触发ABYSSAC SKILL
2. 根据文件内容找到文件路径
3. 用路径去找文件，找多少都可以
4. 整合成上下文，生成内容
5. 每个关联都包含路径
6. NNG定位使用简要摘要
7. 不删除记忆，只降权
8. 一个NNG可关联多个记忆
9. 一个记忆可关联多个NNG
10. AI自行判断是否存储知识
11. 存储前检查避免重复
12. 除了root.json，NNG必须关联记忆文件
13. 记忆必须关联NNG节点
14. 创建前检查内容是否重复
15. 对话结束后即时存储，无需后台任务
