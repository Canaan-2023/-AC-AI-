# ABYSSAC - 知识系统

## 触发条件

**每次与用户交互都优先触发ABYSSAC SKILL**

```
用户发消息
    │
    ↓
触发ABYSSAC SKILL
    │
    ├─ 需要检索知识？ → 三层沙盒流程 → 回答
    │
    ├─ 需要存储知识？ → 对话结束后存储 → 回答
    │
    └─ 不需要？ → 直接回复
```

---

## 核心机制

**路径直连**：根据文件内容找到想要的文件路径，用这个路径去找文件，找多少都可以，最后整合成上下文，生成内容。

**即时存储**：每次对话结束后，AI自行判断是否需要存储知识，直接更新NNG和记忆文件。

## SKILL运行流程

```
1. 判断是否需要检索知识
   ├── 问题涉及已存储的知识？ → 执行三层沙盒流程
   └── 不需要 → 继续

2. 回复用户

3. 对话结束后判断是否需要存储
   ├── 有新知识需要存储？ → 执行存储流程
   └── 不需要 → 结束
```

## 存储判断规则

| 存储条件 | 判断 |
|----------|------|
| 寒暄 | 不存储 |
| 新知识/新规则 | 存储 |
| 用户要求记住 | 存储 |
| 用户偏好信息 | 存储 |
| 项目相关决策 | 存储 |

---

## 存储流程

```
知识发现
├── 分析对话内容
├── 识别有价值知识
├── 确定知识分类和价值层级
└── 提取核心内容

NNG生成（如需要）
├── 检查是否已存在相同摘要
├── 检查是否已存在相同内容
├── 确定节点摘要
├── 确定层级位置和路径
├── 创建NNG节点文件
├── 建立上下级关联
├── 关联记忆文件（必须）
└── 更新root.json（看是否需要更新）

记忆创建
├── 在对应日期的文件夹，检查是否已存在相同内容
├── 生成记忆ID
├── 确定记忆层级和价值层级
├── 确定文件路径
├── 创建记忆文件
├── 写入核心内容
├── 关联NNG节点（必须）
└── 更新NNG节点的记忆摘要
```

| 级别 | 文件位置 | 作用 |
|------|----------|------|
| 一级 | `d:\ABYSSAC\nng\{摘要}.json` | 大的知识领域或概念分类 |
| 二级 | `d:\ABYSSAC\nng\{一级摘要}\{二级摘要}.json` | 一级节点下的具体概念 |
| 三级+ | `d:\ABYSSAC\nng\{一级摘要}\{二级摘要}\{三级摘要}.json` | 更细分的概念 |

## 三层沙盒流程

**Layer 1: NNG导航**
- 读取root.json，根据问题匹配节点摘要，获取路径，读取节点，重复直到获取所有相关记忆文件路径

**Layer 2: 记忆筛选**
- 从NNG节点获取记忆文件路径，读取摘要判断相关性，用路径读取完整记忆内容

**Layer 3: 上下文组装**
- 整合所有NNG节点和记忆文件内容，按相关性排序，组装成上下文，生成回答

## 避免重复机制

**存储前检查：**
1. 检查是否已存在相同摘要的NNG节点
2. 检查是否已存在相同内容的NNG节点
3. 检查是否已存在相同核心内容的记忆
4. 通过读取现有NNG节点的"内容"字段进行比对

**避免重复规则：**
- 相同摘要的NNG节点不重复创建
- 相同核心内容的记忆不重复创建
- 已处理的内容不再处理

## 原则

1. 每次与用户交互都优先触发ABYSSAC SKILL
2. 根据文件内容找到文件路径
3. 用路径去找文件，找多少都可以
4. 整合成上下文，生成内容
5. 每个关联都包含路径
6. NNG定位使用简要摘要
7. 不删除记忆，只降权
8. 一个NNG可关联多个记忆
9. 一个记忆可关联多个NNG
10. AI自行判断是否存储知识
11. 存储前检查避免重复
12. 除了root.json，NNG必须关联记忆文件
13. 记忆必须关联NNG节点
14. 创建前检查内容是否重复
15. 对话结束后即时存储，无需后台任务
