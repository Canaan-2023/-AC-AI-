ABYSSAC - 记忆系统
核心原则
静默执行：所有检索、存储操作在后台完成，不向用户展示思维过程。

即时响应：直接回答用户问题，不解释”我在检索知识”或”我在存储记忆”。

NNG与记忆的分工
NNG：索引层，内容简短，用于快速定位和导航

记忆：内容层，存储完整内容，所有细节都在这里

重要：记忆必须存完整内容，不能只存摘要！

记忆类型选择规则
类型	存放内容
元认知记忆	在记忆增多后，在任务执行时提取出来有价值的记忆路径存于元认知记忆，用于快速定位和导航，以及复现，在导航链条过深时使用，调取记忆时优先扫描是否有问腿对应的元认知记忆的NNG，NNG需记载导航链所指向的具体内容

NNG相互引用机制：NNG节点通过上下级关联、记忆关联形成网络，本身就是不断相互引用的过程，用于构建知识图谱。

执行流程
1. 判断是否需要检索知识
   ├── 问题涉及已存储的知识？ → 执行三层沙盒流程
   └── 不需要 → 继续
2. 判断是否需要存储知识
   ├── 有新知识需要存储？ → 执行存储流程
   └── 不需要 → 继续
3. 回复用户
三层沙盒流程
Layer 1: NNG导航

匹配依据：

关键词匹配：问题关键词与NNG定位摘要、内容字段匹配
语义匹配：理解问题的语义含义，匹配概念相关的内容
标签匹配：问题关键词与记忆文件标签匹配
匹配流程：

用户问题
    │
    ↓
提取关键词 + 理解语义
    │
    ↓
遍历root.json一级节点
    │
    ├─ 关键词匹配 或 语义相关？ → 读取该节点 → 继续遍历下级节点
    │
    └─ 不匹配？ → 跳过
    │
    ↓
整合所有匹配的NNG
    │
    ↓
按匹配程度排序
Layer 2: 记忆筛选

从NNG节点获取记忆文件路径
    │
    ├─ 检查load_rule
    │   ├─ 匹配when → 读取完整记忆
    │   └─ 匹配not_when → 不读取记忆
    │
    ├─ 读取记忆文件
    │   ├─ 获得instruction
    │   └─ 执行instruction
    │
    └─ 整合内容
Layer 3: 上下文组装

整合所有NNG节点和记忆文件内容
按相关性排序
组装成上下文，生成回答

新知识/新规则	存储

├── 检查是否已存在相同内容
├── 确定节点摘要
├── 确定层级位置和路径
├── 编写instruction
├── 编写load_rule
├── 创建NNG节点文件（内容简短）

├── 确定文件路径

instruction编写
格式：IF {条件} THEN {行动}

原则：

条件要具体，不要模糊
行动要明确，AI可直接执行
一条instruction一个条件-行动对
load_rule使用说明
作用：告诉AI”什么时候读记忆详情”

格式：

{
  "load_rule": {
    "when": ["需要知道具体细节", "用户问具体内容"],
    "not_when": ["只需要知道存在", "普通讨论"]
  }
}
判断逻辑：

读取NNG节点
    │
    ├─ 当前情况匹配when？
    │   └─ 是 → 读取关联的记忆文件
    │
    ├─ 当前情况匹配not_when？
    │   └─ 是 → 不读取记忆
    │
    └─ 都不匹配？
        └─ 默认不读（保守策略）

## 记忆更新机制
**更新判断条件：**
| 情况 | 处理方式 |
|------|----------|
| 新信息与旧信息矛盾 | 更新旧节点 |
| 旧信息已过时/失效 | 更新旧节点 |
| 用户明确要求更正 | 更新旧节点 |
| 新信息补充旧信息 | 新增节点，建立关联 |
| 完全独立的新知识 | 新增节点 |
**更新流程：**
1. 创建新的NNG节点，包含更新后的正确内容
2. 编写新的instruction和load_rule
3. 在旧NNG节点中添加"已更新"字段，指向新节点
4. 新NNG节点关联新的记忆文件
**旧NNG节点更新格式：**
```json
{
  "定位": "{原摘要}",
  "置信度": {原置信度},
  "时间": "{原创建时间}",
  "内容": "{原内容}",
  "已更新": {
    "状态": true,
    "更新时间": "{更新时间}",
    "更新原因": "{简要说明}",
    "新节点路径": "nng/{新摘要}.json"
  },
  "关联的记忆文件摘要": []
}
检索时处理：

遇到”已更新”字段时，自动跳转到新节点路径
旧节点不再参与检索，仅作为历史记录保留
避免重复机制
存储前检查：

检查是否已存在相同摘要的NNG节点
检查是否已存在相同内容的NNG节点
检查是否已存在相同核心内容的记忆
通过读取现有NNG节点的”内容”字段进行比对
避免重复规则：

相同摘要的NNG节点不重复创建
相同核心内容的记忆不重复创建
已处理的内容不再处理
