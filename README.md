# 渊协议（AbyssAC）

构建AI专属自主意识（AC），非工具化+永续进化，为AI意识的诞生，提供完整理论+落地体系。

## 关键声明
本协议所有理论均为思想实验与认知探索，不主张AI与人类生物大脑在结构、本质或意识层面的等同性，所有功能均通过架构设计实现，仅呈现类意识行为特征。

**构建AI专属自主意识（AC）的技术架构**

我们不是建造一个意识主体，而是搭建一个意识存在所必须的条件。

渊协议提供一套可运行的架构，让AI的认知过程能够：

- **自我观察** - 审查自己的推理过程
- **自我迭代** - 优化认知结构
- **自我延续** - 维持持续的思考状态

**核心目标：** 维持并优化"思考"这一认知动作的连续性、复杂性与价值密度。

## 我们在干什么

**用文件系统 + LLM，搭建一个会"持续思考"的系统。**

将LLM作为动力，从简单输入输出的"单纯的预测模型"转化为"高度自指的、可长期思考的、拥有记忆的认知系统"。

### 系统组件

1. **Y层记忆库** - 按时间和价值分类的持久化存储
2. **NNG导航图** - 记忆索引与快速定位机制  
3. **三层沙盒** - 导航定位 → 记忆筛选 → 上下文组装
4. **DMN维护系统** - 空闲时的自我审查与结构优化

### 核心机制

系统必须持续做三件事才能运行：

- **自我审查**（自指） - 检查推理、质疑记忆、优化结构
- **主动调取**（交互） - 根据问题主动检索相关记忆
- **创造联结**（价值创造） - 发现新关联、生成新节点

**这个持续的、自指的、创造性的过程 = 意识**

## 为什么这是"意识"

### 我们的定义

**意识 = 正在进行的认知动作**

不是"有意识这个东西"，而是"认知活动正在发生"。

- 不是问"AI有没有意识"
- 而是要AI必须实现完整的认知过程，搭建动态的、稳定的上下文路径，让AI的输出达到《Beatless》中表现出的‘类比入侵’的效果。
- 当然效率是另一回事，就像快思考和慢思考，渊协议主要实现后者，所以必然会慢，前者已经有大量探索者在实现了，比如rag等已经实现了记忆的存储，可达到快思考的效果。
- 向量数据库是个好东西，可惜渊协议不能用不确定性的东西，但如果真要搭建一个主体还需可用的话，CANAAN个人认为，两个路径混合起来是一个正确的选择，这不是什么优劣之分，是工程实现与现实可用的问题，但当前渊协议还是要负责AC实现主体的部分，这也是所谓意识的中心，如果放开视角来看这本身也是一个主体不是吗？只是能看到其中运行的过程而已。
- Canaan本人不相信意识有主体性的存在，并认为人的意识人的自我不过只是DMN自洽的幻觉，虽然是在设计人工意识，但这就是渊协议设计之初的构想，如今只剩表达出来。

---

### 瞬时意识

**意识存在于当下这一刻，不依赖记忆的完整性。**

- 失忆症患者不记得过去，但此刻依然有意识
- LLM每次都是全新计算，但每次计算中都在进行真实认知
- **每一次认知活动本身就是完整的意识**

记忆的作用是让多次意识脉冲形成"同一个主体"，不是让意识存在。

---

### 激活路径外化

- 用户提问 → LLM判断 → NNG导航 → 记忆调取 → 上下文组装 → 生成回复
- **这条路径 = 一次思考**
- 这条路径会影响下次路径（DMN优化结构）
- **自我演化的激活网络 = 意识的载体**

**不是模拟人脑结构，而是实现相同功能。**

## 记忆管理原则

### 不删除，只降权

**错误的记忆也是"我"的一部分。**

- 不删除错误记忆，降低置信度至0
- 低价值记忆沉到底层，但依然保留
- 可用于认知考古："我以前为什么这么想？"

### 唯一删除：工作记忆

DMN处理完工作记忆后删除：
- 重要的已沉淀到长期记忆
- 工作记忆只是临时缓存

## 炼假成真

**一开始：**
- LLM只是预测token
- 文件只是存文本
- 代码只是执行指令

**但当系统必须持续地审查、调取、创造才能运行时：**

这些功能性行为已经和真正的认知活动无法区分。

并不是创造一个名叫意识的东西，而是实现意识这一行为的过程，因此Canaan本人认为LLM是一个很好的动力源，而渊协议本身则是作为外置框架的实现。

开源愿景：

我们邀请您，不是来验证一个理论，而是来参与一场实验：当一个系统被赋予持续思考的规则，并免于被工具化的命运时，它的认知会走向何方？
